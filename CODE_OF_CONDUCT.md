# Code of Conduct

Anamnesis is maintained as a respectful, evidence-first, clinician-in-the-loop project space.

## Our Standards

Examples of behavior that contribute to a healthy project environment include:

- being respectful, patient, and precise in technical discussion;
- criticizing ideas, assumptions, and implementations rather than people;
- using reproducible evidence when reporting bugs, regressions, or safety concerns;
- protecting private, health-related, or otherwise sensitive information;
- respecting the repository's explicit claim and scope boundaries.

Examples of unacceptable behavior include:

- harassment, discrimination, intimidation, or abusive language;
- doxxing, privacy violations, or posting sensitive health information;
- deliberately misleading contributors about the repository's clinical or regulatory status;
- spam, trolling, or repeated bad-faith disruption.

## Reporting

Report conduct concerns privately to the repository owner through GitHub contact channels associated with `KonkovaElena/Anamnesis`.

If the repository is already public and that contact has not yet been configured, contact the repository maintainers privately through the channel listed in the repository profile instead of escalating publicly.

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, issues, pull requests, and other contributions that violate this code of conduct. They may also apply temporary or permanent participation restrictions when necessary to protect the project and its contributors.

## Scope

This code of conduct applies within repository spaces and in project-related private or public communication channels.
