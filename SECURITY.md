# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| 1.x | Yes |

## Reporting A Vulnerability

Do not use public issues for security vulnerabilities.

Preferred reporting path after the repository is created on GitHub:

1. Use GitHub private vulnerability reporting for the repository when it is enabled: `https://github.com/KonkovaElena/Anamnesis/security/advisories/new`.
2. If private reporting is not enabled yet, contact the repository owner privately through GitHub rather than opening a public issue.

When reporting a vulnerability, include:

- affected version or commit;
- reproduction steps or a minimal proof of concept;
- impact assessment;
- any proposed mitigation if known.

## Response Targets

- Initial acknowledgement target: 3 business days
- Status update target: 7 business days

## Disclosure

Please avoid public disclosure until maintainers have had a reasonable chance to investigate, mitigate, and communicate a coordinated fix.
