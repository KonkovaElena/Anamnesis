# Support

Use this file to choose the right support path for Anamnesis.

## Where To Ask For Help

- Bug reports: open a GitHub issue with a reproducible example, expected behavior, actual behavior, and logs if available.
- API or integration questions: open a GitHub issue and describe the endpoint, payload shape, and desired workflow.
- Documentation gaps: open a docs issue or pull request with the missing page, claim, or broken link.

## Security Reports

Do not open public issues for vulnerabilities, secrets, or unsafe deployment findings.

- Follow the instructions in [SECURITY.md](SECURITY.md).
- If you are unsure whether something is a vulnerability, treat it as security-sensitive first.

## What Maintainers Need To Triage Faster

- Exact commit or release tag.
- Node version and operating system.
- Minimal reproduction steps.
- Redacted logs or request/response samples where relevant.

## Scope Reminder

Anamnesis is a consultation-intelligence backend, not a diagnostic or triage product. Questions that assume medical advice, treatment recommendations, or emergency decision support are out of scope for repository support.
